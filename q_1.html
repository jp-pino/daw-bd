Cuestionario 1: BD vs DBMS

En esta sección se responderán las preguntas del cuestionario de bases de datos.

Preguntas

...y respuestas

¿Qué es entonces una base de datos - DB?

Una base de datos es una colección de información relacionada[5] organizada para proporcionar una acceso efeciente a la misma[1]. De esta manera una base de datos no es más que la información almacenada de manera organizada que se trata como unidad para acceder a información relacionada[2]. Al utilizar una base de datos electrónica, los usuarios buscan automatizar el acceso a la información.

¿En qué casos es conveniente usar bases de datos?[3]

  • Cuando la información almacenada es muy grande.
  • Más de una persona accesará la información simultaneamente.
  • Cuando la información va a ser utilizada en otras aplicaciones.
  • Cuando la seguridad de la información es importante.
  • Cuando se quieren definir permisos de acceso a la información.

¿Qué es un sistema de gestión de base de datos - DBMS?

Un sistema de gestión de base de datos (por sus siglas en inglés DataBase Management System) es un sistema computarizado de manjejo de datos que les da a los usuarios la facilidad de manipular la información en la base de datos o su estructura[4].

  • Enlista y explica las funciones/responsabilidades que tienen los DBMS.

    Los DBMS resuelven las siguientes problemáticas6.

  • Redundancia e inconsistencia

    El DBMS se asegura de que la información almacenada no esté duplicada y que todos los usuarios puedan acceder a la información mas actualizada.

  • Dificultad en el acceso

    Debido a las relaciones implementadas en el sistema es muy facil para el usuario acceder a información específica organizada de la manera que le sea mas conveniente.

  • Aislamiento

    Al implementarse bases de datos relacionales, los archivos registrados en el sistema pueden ser accesados por el mismo usuario de manera simultánea sin preocuparse de buscar los archivos.

  • Problemas de integridad

    Al implementar restricciones de consistencia, se asegura que la información almacenada y por ser registrada siga la estructura de las tablas definidas en el sistema.

  • Problemas de atomicidad

    El DBMS se asegura de que la información se restaure después de un fallo de sistema, para asegurar su seguridad y consistencia.

  • Problemas de seguridad

    El DBMS permite la creación de usuarios y roles para restringir el acceso a la información o su modificación.

  • Problemas de acceso concurrente

    El DBMS maneja las modificaciones concurrentes supervisando los cambios realizados. De esta manera intenta asegurar al usuario que la información que recibe está acutalizada.

Ejemplifica y justifica en que proyecto de los que has realizado hubiera sido conveniente utilizar una base de datos.

Para el "Laboratorio de Microcontroladores" hice un control de acceso basado en un módulo lector de huellas dactilares. Las entradas y salidas eran registradas en un archivo de texto. Hubiera sido conveniente tener una base de datos de usuarios registrados para estos manejar un mayor número de usuarios.
