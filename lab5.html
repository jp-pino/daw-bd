¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Ambos lenguajes comparten algunas semejanzas: son orientados a objetos, pueden correr en servidores, en navegadores web, y pueden usar frameworks y librerias[2][3]. Sin embargo, son más las diferencias que hay entre ellos. Partiendo de que Java es un lenguaje de programación orientado a objetos y que JS es un lenguaje de scripting orientado a objetos podemos derivar otras diferencias. Mientras que Java se compila y corre dentro de un JVM o navegador, JS es interpretado y solo puede funcionar dentro de un navegador. También cabe destacar que JavaScript está diseñado para dar interactividad a documentos HTML mientras que las aplicaciones Java cubren necesidades mucho mas amplias.
