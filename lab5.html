¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Porque le ahorra tiempo al usuario. La página funcionaría igual, pero habria que esperar una respuesta negativa del servidor antes de darse cuenta del error en el formulario.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Desactivando JavaScript en el servidor.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Básicamente por ahorrarle tiempo al usuario. Sin embargo, el sistema debe validar de cualquier manera los datos ingresados al formulario ya en el lado del servidor por seguridad.

¿Cuáles son las diferencias entre los posibles valores de la propiedad position?[1]

  • static: los elementos aparecen en el orden del documento.
  • absolute: el elemento es posicionado respecto a su primer ancestro no estático (static).
  • fixed: el elemento es posicionado respecto a la ventana.
  • relative: el elemento es posicionado respecto a su posición normal.
  • sticky: el elemento es posicionado respecto a la posición del scroll del usuario.
  • initial: configura a default (visible).
  • inherit: heredar propiedad del elemento padre.

¿Cuáles son los valores estándar para la propiedad visibility?[2]

  • visible: el elemento es visible (Valor Estándar).
  • hidden: el elemento es invisible y ocupa espacio.
  • collapse: para tablas. Esconde filas o columnas sin afectar la tabla.
  • initial: configura a default (visible).
  • inherit: heredar propiedad del elemento padre.

¿Qué es el zIndex y para qué sirve?[3]

El zIndex especifíca el orden en que aparecen los elementos en patalla al estar unos sobre otros.
